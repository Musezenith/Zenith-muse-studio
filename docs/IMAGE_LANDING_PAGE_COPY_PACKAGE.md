### Problem section
Section title:
- **Image generation is fast. Image decisions are still messy.**

Body copy:
- Most teams can generate images quickly, but still lose time deciding what to keep, explaining why, and handing results off cleanly.
- Common issues:
  - Prompt inconsistency across team members
  - Slow compare-and-approve cycles
  - Missing run history and unclear final decisions
  - Fragile file/link handling in review

### Solution section
Section title:
- **Musezenith turns generation into a production-ready decision workflow**

Body copy:
- Instead of isolated prompts, you get a structured path from brief to approved output:
  - Build prompt package in PromptLab
  - Generate variants with seed and model controls
  - Review, compare, and mark final outputs
  - Re-run from history when needed
  - Export one clean bundle for review and handoff

### Workflow section (must match shipped flow)
Section title:
- **How it works**

Step copy:
1. **PromptLab**: Enter brief, choose preset/model/variants/seed policy.
2. **Generate**: Create image variants and review Positive/Negative/Params/QC/JSON/Score tabs.
3. **Archive**: Every run is saved for traceable history.
4. **Compare**: Place two runs/images side-by-side (Left/Right).
5. **Final selection**: Mark run final and/or mark specific images final.
6. **Re-run**: Generate fresh outputs from an archived payload.
7. **Export**: Download a bundle with prompt, result, metadata, and asset references.

### Objection handling section
Section title:
- **Common questions before teams switch**

Q/A:
- “We already use image generators.”
  - Musezenith adds the decision and handoff workflow after generation.
- “What if generation fails?”
  - Retry/cancel and run-state tracking are built in.
- “What if links break?”
  - Asset handling is remote-first with fallback behavior per asset.
- “How do we share outcomes?”
  - Export bundle includes prompt/result/metadata/asset references.
